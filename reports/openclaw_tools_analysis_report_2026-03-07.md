### 1️⃣ ORCHESTRATION (오케스트레이션)

#### 1.1 TenacitOS ⭐ P1 권장
- **개발자**: carlosazaustre (GitHub 392 stars)
- **기술 스택**: Next.js, React 19, Tailwind CSS v4
- **핵심 기능**: OpenClaw 인스턴스 실시간 대시보드, 설정/에이전트/세션/메모리/로그 직접 읽기
- **사용 사례**: 
  - 다중 OpenClaw 인스턴스 중앙 관리
  - 실시간 모니터링 및 제어
  - 개발자 친화적 UI
- **잠재적 문제**:
  - React 19 최신 버전 의존성 → 안정성 우려
  - OpenClaw 워크스페이스 낮은 수준 접근 권한 필요
  - UI 기반 관리 vs CLI 선호 개발자 갈등
- **도입 권장도**: ⭐⭐⭐⭐ (4/5)

#### 1.2 Robsannaa's Mission Control
- **개발자**: robsannaa (오픈소스)
- **핵심 기능**: 로컬 GUI, 에이전트 채팅/관리, Tailscale 통합
- **차별점**: 기술 비유저 친화적, agentbay.space 배포 옵션
- **사용 사례**:
  - 비개발자 팀원용 관리 인터페이스
  - 원격 에이전트 접근 (Tailscale)
- **잠재적 문제**:
  - TenacitOS와 기능 중복
  - 로컬 GUI vs 웹 기반 선호도 문제
- **도입 권장도**: ⭐⭐⭐ (3/5) - 이미 TenacitOS 도입 시 중복

#### 1.3 Autensa
- **공식 사이트**: autensa.com
- **핵심 기능**: AI 계획 → 전문 에이전트 배치 → 실시간 추적
- **차별점**: 인터랙티브 Q&A 기반 계획, Kanban 보드
- **사용 사례**:
  - 복잡한 멀티에이전트 워크플로우
  - 작업 큐 관리
- **잠재적 문제**:
  - 프로덕션 준비도 불명확 (Alpha 소프트웨어)
  - Builderz Mission Control과 유사성
- **도입 권장도**: ⭐⭐⭐ (3/5)

#### 1.4 ClawController ⭐ 공식 도구
- **공식 사이트**: clawcontroller.com
- **개발사**: OpenClaw 공식
- **핵심 기능**: 실시간 모니터링, 작업 오케스트레이션, 직접 에이전트 채팅

  - OpenClaw 네이티브 통합
- **잠재적 문제**:
  - 서드파티 대시보드와 비교해 유연성 제한
  - 비용 (공식 도구의 경우 상용 라이선스 가능)
- **도입 권장도**: ⭐⭐⭐⭐ (4/5) - 공식 지원 필요 시

### 2️⃣ INFRASTRUCTURE (인프라)

#### 2.1 ClawControl ⭐ P2 권장
- **개발사**: OpenClaw
- **핵심 기능**: 중앙 집중식 클라우드 모니터링, 성능/비용/로그 시각화, AI 기반 최적화 제안
- **사용 사례**:
  - 대규모 에이전트 배포 환경
  - 비용 최적화 필요 조직
  - 클라우드 중심 운영
- **잠재적 문제**:
  - Clawmetry(묘니터링)와 기능 중복
  - 클라우드 의존성 → 온프레미스 환경 제한
- **도입 권장도**: ⭐⭐⭐⭐ (4/5)

#### 2.2 Clawd Cursor ⭐ P2 권장
- **공식 사이트**: clawdcursor.com
- **핵심 기능**: 데스크톱 제어 스킬, 화면 인식, 마우스/키보드 제어, 자율 작업 완료
- **사용 사례**:
  - GUI 기반 레거시 시스템 자동화
  - 브라우저 외 데스크톱 앱 제어
  - RPA (Robotic Process Automation) 대체
- **잠재적 문제**:
  - **보안 리스크**: 데스크톱 제어 = 높은 권한 필요
  - 오작동 시 시스템 손상 가능성
  - PinchTab(브라우저)와 역할 구분 필요
- **도입 권장도**: ⭐⭐⭐⭐ (4/5) - 보안 가드레일 필수

#### 2.3 Clawzempic ⭐⭐⭐ P0 필수
- **공식 사이트**: clawzempic.ai
- **핵심 가치**: **LLM API 비용 70-93% 절감**, 동일 모델/품질
- **기술**: 드롭인 LLM 프록시, 라우팅, 캐싱, 메모리, 윈도잉
- **설치**: 30초, `/learn @openclaw/clawzempic`
- **사용 사례**:
  - **모든 OpenClaw 사용자 필수**
  - 고비용 API 사용 팀 (GPT-4, Claude 등)
  - 비용 최적화 우선 조직
- **잠재적 문제**:
  - 프록시 추가 = 지연 시간 증가 (5ms 미만 주장)
  - 캐싱 적중률에 따른 효과 편차
  - 서비스 장애 시 단일 장애점
- **ROI 분석**: 월 API 비용 100만원 → 7-30만원 절감 (연간 840만원+ 절감)
- **도입 권장도**: ⭐⭐⭐⭐⭐ (5/5) - **즉시 도입**

#### 2.4 PinchTab ⭐ P2 권장
- **GitHub**: pinchtab/pinchtab (12MB Go 바이너리)
- **핵심 기능**: HTTP API 브라우저 제어, 멀티 인스턴스 오케스트레이션, 스텔스 인젝션
- **기술**: Chrome DevTools Protocol (CDP), HTTP in / CDP out
- **사용 사례**:
  - WiseReport 스크레이핑 (현재 사용 중)
  - 웹 데이터 추출 자동화
  - 멀티 계정 브라우저 관리
- **잠재적 문제**:
  - Playwright/Puppeteer와 경쟁
  - OpenClaw 네이티브 브라우저 툴과 중복 가능성
  - 스텔스 기능의 윤리적 문제
- **도입 권장도**: ⭐⭐⭐⭐ (4/5) - 현재 Playwright 대체 검토 가능

#### 2.5 VisionClaw
- **GitHub**: sseanliu/VisionClaw
- **핵심 기능**: Meta Ray-Ban 스마트글래스용 실시간 AI 어시스턴트
- **기술**: 음성 + 비전 + 에이전틱 액션 (Gemini Live + OpenClaw)
- **사용 사례**:
  - 핸즈프리 현장 작업 지원
  - 시각적 데이터 입력 ("이게 뭐야?")
  - 이동 중 AI 어시스턴스
- **잠재적 문제**:
  - 하드웨어 의존성 (Meta Ray-Ban 필요, $300+)
  - 배터리 수명 제한
  - 프라이버시 우려 (칠레 녹화 등)
- **도입 권장도**: ⭐⭐ (2/5) - 특수 사용 사례 제한적

#### 2.6 VoxClaw
- **플랫폼**: macOS 메뉴 바 앱
- **핵심 기능**: 고품질 음성으로 OpenClaw 에이전트 대화
- **사용 사례**:
  - 음성 선호 사용자
  - 멀티태스킹 중 빠른 명령
- **잠재적 문제**:
  - macOS 전용 (Windows/Linux 미지원)
  - Telegram/WhatsApp 음성 메시지와 중복
- **도입 권장도**: ⭐⭐⭐ (3/5) - macOS 사용자만

**🎯 Infrastructure 카테고리 결론**:

- **사용 사례**:
  - **모든 OpenClaw 배포 필수**

- **잠재적 문제**:
  - 오탐 (false positive) 가능성
  - 설정 복잡성
  - 성능 오버헤드 (5ms 미만 주장)
- **도입 권장도**: ⭐⭐⭐⭐⭐ (5/5) - **즉시 도입**
